# JEPA-HT: Hierarchical Text Architecture

## Overview

JEPA-HT is a self-supervised hierarchical text representation system built on the Joint-Embedding Predictive Architecture (JEPA) paradigm. It learns compositional structure in language from the bottom up using a stack of locally trained JEPA modules. No global backpropagation passes through the hierarchy — each level is trained independently with its own local objective.

Key contributions:

- **Novel JEPA training losses:**
    - A triplet loss that prevents JEPA from collapsing to a constant embedding for all inputs.
    - Anti-toward-zero mitigation that prevents the model from converging to near-zero outputs.
- Solves the representational lag present in standard causal transformers.
- Enables compartmentalized training of neural networks.
    - Training time to convergence grows exponentially with parameter count. Training many small networks sequentially mitigates this exponential growth.
    - Each module is independently reusable — the same module can serve sentence embedding and next-token prediction.
- Module outputs can be concatenated and used as input to subsequent modules, allowing the effective context window to grow without retraining the full network.
- Once text embeddings are sufficiently expressive, additional transformers can operate on embeddings of text spans rather than individual tokens, eliminating the exponential growth of context length.


Key properties:

- No global backpropagation across levels
- Context window grows with depth, not as a fixed hyperparameter
- New levels can be added after training without retraining existing levels
- Each level is independently deployable and interpretable


## Cross-Attention Training vs. Self-Attention Inference

Standard causal transformers suffer from a representational lag problem: at position `t`, layer `l` only has access to the last layer's residual stream of previous tokens. This is suboptimal during inference, as the information from prior tokens always reflects a prediction — one that was subsequently sampled from. The state of previous tokens is therefore strictly less informative than it theoretically needs to be.

JEPA-HT addresses this by running the model twice: once for prediction and once for context generation.

This is achieved using a causal mask that is one step behind, with the prediction stream cross-attending to the context stream. This gives the generator access to fully-formed representations of the real context at every layer.

**At training time**, the full context generation runs as a single forward pass, keeping training efficient.

**At inference time**, the model first generates a prediction. A sample is then drawn from that prediction and used to generate the correct context state.

Each `DoubleTransformerBlock` is structured to support this:
```
input_mlp → layer1 (cross-attn during training / self-attn at inference) → layer2 (always self-attn) → output_mlp
```
The `input_mlp` runs first on every forward pass, allowing the block to detect and adapt to whether its input is a prediction or real data. A small fraction of positions — 2 out of every 256 — use the real input token instead of the null token, preventing the model from ignoring real token input entirely at inference time.


## Training Losses

Each layer is trained with a per-layer triplet objective.

### Attract

MSE between the predictor output and the clean target latent. This is the core JEPA signal, pushing the latent space to be maximally useful for representation.

### Toward-Zero Mitigation

Minimising MSE alone creates an incentive to predict near-zero values, since zero is close to the mean of any zero-centred distribution. The toward-zero component measures the squared mean of the prediction error across the batch and sequence — quantifying the systematic bias in the predictor. This bias is stripped from the gradient via gradient centering, so the model is only rewarded for matching the *relative pattern* of target values, not their absolute magnitude.

### Repel

Without a repulsion term, the attract loss can be satisfied by collapsing all representations to the same non-zero constant. Two cosine-distance terms prevent this by pushing representations away from corrupted inputs:

- `repel`: cosine distance between the predictor output and the corrupt latent
- `repel_tc`: cosine distance between the clean target and the corrupt latent

Corrupted inputs are generated by sampling random bytes guaranteed to differ from the real input at every position. Together, the repel terms act as a contrastive signal at the latent level, ensuring clean and corrupt representations remain well-separated.
